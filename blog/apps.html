What do they know?

An analysis of mobile application data collection

In the past few months, I have seen a great deal of media attention about how the TikTok application collects a variety of information about their user's devices and the implications that collection has to personal security. While I have seen several suspect things about the data collection that TikTok performs, it is by no means the only mobile application that collects an unreasonable amount of data. I have been doing mobile application penetration testing for several years, and mobile phone data collection is almost obligatory. This data is collected mainly for internal company use or sold to advertisers.

In this blog post, I have created a table that outlines the types of data that some of the most popular mobile applications in the US collect. This data is collected by the applications every time that you open the application, whether or not you actually use the application. From this table, you can see that some of the worst offenders are Uber, Duolingo, and Chick-fil-a. The worst data types in my mind are device name, which by default contains the user's first name, the device's IP address, and data that together could be used to identify the activities of the user, such as accelerometer, brightness, phone orientation, and device charging status.

For privacy reasons, I cannot release the exact outgoing requests that contained this data, but if you would like to verify the information contained within this blog, please contact me through email or on twitter.
